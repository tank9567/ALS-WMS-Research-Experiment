### 1. 시스템 구성도 시안 (System Architecture)

이 그림은 팀원들이 어떻게 공유된 로직을 중심으로 AI와 협업하는지를 한눈에 보여줍니다. (PPT나 화이트보드에 그릴 때 참고하세요.)

* **핵심 레이어 설명:**
1. **Shared Logic Repository (중앙 저장소):** 팀이 합의한 비즈니스 로직(Atomic Logic Sheets)이 마크다운 형태로 저장된 곳입니다. 프로젝트의 '헌법' 역할을 합니다.
2. **AI Mediator (중재자 에이전트):** 개발자의 프롬프트와 중앙 저장소의 로직을 실시간으로 비교하는 엔진입니다.
3. **Conflict Detection (충돌 탐지기):** 사용자의 요청이 기존 로직("A이면 B")과 다를 경우("A인데 C로 해줘") 즉각적으로 팝업이나 경고를 띄우는 필터입니다.
4. **Feedback Loop:** "설계를 변경하시겠습니까?" 혹은 "실수를 수정하시겠습니까?"라고 묻는 인간-AI 상호작용 단계입니다.

### 2. 예비 실험 결과 보고서 (PoC Report) 초안

교수님께 "제가 직접 테스트해 보니 이런 결과가 나왔습니다"라고 말씀드리기 위한 요약본입니다.

#### [보고서: 비즈니스 로직 주입 및 앵커링의 효과성 검증]

**1. 실험 개요**

* **대상:** 복잡한 비즈니스 규칙(예: 특정 조건에서의 데이터 가공 로직 5개 이상)이 포함된 모듈 개발.
* **비교군 (A):** 일반적인 프롬프트 방식 (설계 문서 없이 구두로 지시).
* **실험군 (B):** 본 연구에서 제안하는 **H-BLI & Anchoring** 방식 적용.

**2. 주요 실험 결과**
| 평가 항목 | 일반 방식 (A) | 제안 방식 (B) | 비고 |
| :--- | :---: | :---: | :--- |
| **로직 구현 정확도** | 약 65% | **95% 이상** | 복잡한 예외 처리에서 차이 발생 |
| **할루시네이션 발생율** | 높음 (임의 필드 생성) | **매우 낮음** | DDL 및 로직 시트 참조 효과 |
| **논리 충돌 탐지** | 불가능 (AI가 무조건 동조) | **즉시 탐지 및 경고** | 사용자의 변심/실수 차단 확인 |

**3. 정성적 결론**

* 단순 지시 방식은 프로젝트가 진행될수록 AI가 이전 맥락을 잊거나 사용자의 잘못된 지시에 동조하여 전체 로직이 오염되는 현상이 발생함.
* **'논리 앵커링'**을 적용했을 때, AI가 오히려 개발자의 실수를 바로잡아주는 **'역감사(Reverse Auditing)'** 기능이 작동함을 확인하였음.
