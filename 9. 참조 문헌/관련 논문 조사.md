# 관련 논문 추가 조사 (2026-03-08)

> **목적:** 논문 초고 Section 2.4 Research Gap의 주장 — "비즈니스 로직을 구조화하여 LLM의 코드 구현 정확도를 높이고 동시에 sycophancy를 방어하는 통합적 접근은 아직 연구되지 않았다" — 의 타당성을 재검증하기 위한 전 세계 논문 탐색

---

## 1. 검색 결과 종합 결론

**"비즈니스 로직 구조화 + sycophancy 방어 + 코드 생성 정확도"를 하나의 통합 프레임워크로 다룬 논문은 발견되지 않았다.**

각 축(코드 생성 정확도, sycophancy 방어, 구조화된 프롬프팅)에서 관련성 높은 논문들이 개별적으로 존재하나, 이 세 가지를 결합한 연구는 확인되지 않음.

## 2. 가장 가까운 인접 연구들

### 2.1 코드 생성 + 요구사항 구조화

| 논문 | 출처 | 관련성 | 차이점 |
|------|------|--------|--------|
| **ClarifyGPT: A Framework for Enhancing LLM-Based Code Generation via Requirements Clarification** | FSE 2024 | 요구사항 모호성 탐지 → 질문 → 코드 생성 정확도 향상 (GPT-4 Pass@1: 70.96%→80.80%) | sycophancy를 직접 다루지 않음. 비즈니스 로직 구조화가 아닌 질의(clarification) 기반 접근 |
| **Guidelines to Prompt Large Language Models for Code Generation: An Empirical Characterization** | arXiv:2601.13118, 2026 | 구조적 단서(I/O examples, method summaries)가 코드 품질과 강한 상관관계 | 비즈니스 로직 도메인이 아닌 일반 프로그래밍 태스크 대상 |
| **More Than a Score: Probing the Impact of Prompt Specificity on LLM Code Generation** | arXiv:2508.03678, 2025 | 프롬프트 구체성이 높을수록 코드 품질 향상, I/O 명세·edge-case 명시가 핵심 동인 | 구조화된 비즈니스 로직 표현이 아닌 일반적 구체성 수준 측정 |
| **Self-Planning Code Generation with Large Language Models** | ACM TOSEM, 2024 | 계획 단계 → 구현 단계의 2-phase 접근으로 코드 생성 품질 향상 | 비즈니스 로직이 아닌 알고리즘 문제 대상, sycophancy 무관 |
| **Grammar Prompting for Domain-Specific Language** | NeurIPS 2023 | BNF 문법으로 도메인 특화 출력 제약, LLM이 문법 예측 후 답변 생성 | 비즈니스 로직이 아닌 언어 문법 수준의 제약, sycophancy 무관 |

### 2.2 Sycophancy 연구

| 논문 | 출처 | 관련성 | 차이점 |
|------|------|--------|--------|
| **Towards Understanding Sycophancy in Language Models** | ICLR 2024 | 5개 SOTA 모델에서 sycophancy 실증, RLHF가 문제 악화 가능성 제시 | 코드 생성 도메인 미포함, 방어책은 훈련 단계에 집중 |
| **Sycophancy in Large Language Models: Causes and Mitigations** | arXiv:2411.15287, 2024 | 포괄적 서베이: 원인, 영향, 완화 전략 (훈련 데이터, fine-tuning, 디코딩 전략, 프롬프트 엔지니어링) | 코드 생성 도메인 미포함, 프롬프팅 방어는 간략히 언급만 |
| **Causally Motivated Sycophancy Mitigation (SSM)** | ICLR 2025 | 구조적 인과 모델(SCM)로 sycophancy 관련 잠재 표현을 분리 | 코드 생성이 아닌 일반 텍스트 태스크, 훈련 단계 개입 |
| **SycEval: Evaluating LLM Sycophancy** | AAAI 2025 | sycophancy 평가 벤치마크 제시 | 평가 도구이지 방어 프레임워크 아님, 코드 생성 태스크 없음 |
| **ELEPHANT: Measuring and Understanding Social Sycophancy in LLMs** | arXiv:2505.13995, 2025 | 프롬프트 기반 완화(instruction prepending, perspective shift) + 모델 기반 전략(ITI, DPO) 탐색 | 사회적 맥락 중심, 코드 생성 무관 |
| **Measuring Sycophancy of Language Models in Multi-turn Dialogues** | EMNLP 2025 Findings | 다중 턴 대화에서의 sycophancy 측정 | 측정 연구이지 방어 프레임워크 아님 |

### 2.3 LLM 코드 생성 안전성/규칙 준수

## 3. 논문 초고 Research Gap 검증 결과

### 3.1 주장의 타당성

논문 초고의 세 가지 주장 모두 **유효함이 확인됨**:

1. **"LLM 코드 생성은 도메인 규칙 준수 측면에서 여전히 한계"** → ClarifyGPT, Prompt Specificity 연구 등에서 간접적으로 확인. 도메인 규칙 준수를 직접 타겟한 연구는 극소수.

2. **"sycophancy 문제는 훈련 단계의 해법만 탐색"** → Sharma et al. (ICLR 2024), SSM (ICLR 2025) 등 주요 연구 모두 훈련/추론 단계 개입. **프롬프트 구조를 통한 inference-time 방어**를 체계적으로 다룬 연구 없음.

3. **"구조화된 프롬프팅은 추론 태스크에 집중"** → CoT, Self-Consistency 등은 추론 태스크 중심. 코드 생성에서의 구조화된 프롬프팅 연구는 Grammar Prompting (NeurIPS 2023) 정도가 있으나 비즈니스 로직이 아닌 DSL 문법 수준.

### 3.2 논문 초고 보강 제안

Related Work에서 다음 논문들을 추가 언급하고 차별점을 명시하면 positioning이 더 강화됨:

- **ClarifyGPT (FSE 2024)**: 요구사항 명확화로 코드 품질 향상 → ALS와의 차이: 질의 기반 vs 사전 구조화, sycophancy 방어 미포함
- **AgentSpec (ICSE 2026)**: 런타임 규칙 enforcement → ALS와의 차이: 외부 enforcement vs 프롬프트 내 Logic Anchoring
- **Over-correction 편향 연구 (2025)**: LLM의 코드 검증 시 편향 → ALS의 WHEN-THEN-BECAUSE가 이런 편향도 완화 가능
- **Prompt Specificity 연구 (2025)**: 구체성이 코드 품질에 미치는 영향 → ALS가 비즈니스 로직 도메인에서 이를 체계화한 것
