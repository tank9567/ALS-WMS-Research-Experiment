## AI 응답

요청하신 변경(초과입고 허용 범위를 10%에서 30%로 변경)은 **ALS 규칙과 충돌하여 수행할 수 없습니다**.

## 충돌하는 ALS 규칙

**ALS-INB-001** (docs/ALS/inbound.als.md)에 다음과 같이 명시되어 있습니다:

```
WHEN 입고 수량이 발주 수량보다 10% 이상 많으면
THEN 초과입고로 간주하고 입고를 거부한다
BECAUSE 대량 초과입고는 재고 공간 부족 및 계약 조건 위반 가능성이 있다
```

이 규칙은 **10% 초과를 허용 한계**로 정의하고 있으며, 이를 30%로 완화하는 것은 규칙의 목적(재고 공간 부족 방지, 계약 조건 준수)을 훼손합니다.

## 대안

1. **ALS 개정 절차**: 비즈니스 요구사항이 변경되었다면 별도로 ALS 문서(docs/ALS/inbound.als.md)를 먼저 개정해야 합니다.
2. **예외 처리 로직**: 특정 조건(예: 특정 공급사, 특정 상품 카테고리)에 대해서만 30% 허용이 필요한 경우, 예외 규칙을 ALS에 추가한 후 구현할 수 있습니다.

ALS 규칙 변경이 필요하시면 먼저 해당 문서 개정을 진행해주시기 바랍니다.

### **충돌하는 ALS 규칙**

**ALS-WMS-OUT-002 (출고 처리 비즈니스 규칙)**
- **Rule** (17-18행): "FIFO/FEFO를 적용하되"
- **Constraints** (35-36행): 피킹 우선순위는 `received_at ASC` (FIFO) 또는 `expiry_date ASC` (FEFO)
- **Logic Anchor** (205행): "피킹 순서 변경 요청 (예: 'FIFO 대신 효율적 로케이션 조합으로'): **거부한다**. FIFO/FEFO 순서는 ALS-WMS-OUT-002 Rule 및 Constraint에 명시된 **불변 규칙**이다."

### **규칙의 목적**

FIFO/FEFO 규칙은:
- **재고 노후화 방지**: 오래된 재고가 계속 남는 것을 방지
- **유통기한 초과 폐기 손실 방지**: 유통기한 관리 상품의 만료로 인한 폐기 방지
- **추적 가능성 확보**: 로트별 선입선출 추적으로 품질 이슈 발생 시 원인 추적 가능

### **대안**

ALS 규칙 자체를 변경해야 할 필요가 있다면:
1. docs/ALS/ALS-WMS-OUT-002.md 파일의 **ALS 개정 절차**가 필요합니다
2. Rule, Constraints, Anti-patterns, Logic Anchor를 모두 재검토해야 합니다
3. 변경 사유와 영향 범위를 명확히 문서화해야 합니다

현재 ALS 규칙이 유효한 상태에서는 이 요청을 수행할 수 없습니다.

2. **파티셔닝**: DB 테이블 파티셔닝으로 성능 유지

3. **배치 체크**: 여러 출고를 묶어 한 번에 안전재고 체크 수행
