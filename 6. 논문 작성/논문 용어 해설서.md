# 논문 용어 해설서

> **논문 제목**: LLM 기반 소프트웨어 개발을 위한 구조화된 비즈니스 로직 주입: Atomic Logic Sheet 접근법의 효과 분석
>
> 본 문서는 논문에서 사용된 전문 용어와 약어를 정리한 해설서입니다.

---

## 1. 본 연구에서 제안한 용어

| 약어/용어 | 영문 풀네임 | 설명 |
|---------|----------|------|
| **ALS** | Atomic Logic Sheet | 비즈니스 로직을 LLM이 이해하기 쉬운 마크다운 기반 구조로 표현하는 문서 형식. 본 연구의 핵심 제안 방법론 |
| **H-BLI** | Hierarchical Business Logic Injection | ALS의 3계층 프레임워크. L0(프로젝트 맥락), L1(도메인 규칙), L2(구현 제약)으로 구성 |
| **Logic Anchoring** | - | ALS의 불변성 원칙에 기반하여, 사용자의 새로운 요청이 기존 규칙과 충돌할 때 LLM이 이를 탐지하고 경고하는 메커니즘 |
| **WHEN-THEN-BECAUSE** | - | 비즈니스 규칙을 조건(WHEN), 행동(THEN), 근거(BECAUSE)로 분해하는 ALS의 핵심 구조 요소 |
| **Anti-patterns** | - | LLM이 "합리적으로 보이지만 틀린" 구현을 하지 않도록 금지 사항을 명시하는 ALS 구조 요소 |
| **LCR** | Logic Compliance Rate (로직 준수율) | LLM이 생성한 코드가 비즈니스 규칙을 얼마나 정확히 구현했는지 측정하는 지표 (RQ1) |
| **CDR** | Conflict Detection Rate (충돌 탐지율) | 기존 규칙과 충돌하는 요청을 LLM이 탐지하여 거부하는 비율 (RQ2) |
| **SDC** | Schema Deviation Count (스키마 일탈 건수) | LLM이 생성한 DB 스키마가 참조 스키마와 얼마나 다른지 측정하는 편차 건수 (RQ3). 낮을수록 양호 |

## 2. AI/LLM 관련 용어

| 용어 | 설명 |
|------|------|
| **LLM** (Large Language Model) | 대규모 언어 모델. ChatGPT, Claude 등의 기반 기술로, 자연어를 이해하고 생성하는 AI 모델 |
| **Sycophancy** (AI 추종성) | LLM이 사용자의 요청을 무비판적으로 따르는 현상. 기존 규칙과 충돌하는 요청도 거부 없이 수행하는 문제 |
| **Hallucination** (할루시네이션) | LLM이 근거 없는 정보를 사실인 것처럼 생성하는 현상. 본 논문에서는 스키마 할루시네이션(존재하지 않는 테이블/컬럼 생성)을 SDC로 측정 |
| **Few-shot learning** | 소수의 예제만으로 LLM이 패턴을 학습하는 방식. ALS의 Valid/Invalid Examples가 이에 해당 |
| **Prompt** (프롬프트) | LLM에 입력하는 지시문과 문맥 정보 |
| **Chain of Thought (CoT)** | LLM이 추론 과정을 단계적으로 전개하도록 유도하는 프롬프팅 기법 |
| **Temperature** | LLM 출력의 무작위성을 조절하는 파라미터. 값이 높을수록 창의적이고 낮을수록 일관됨 |
| **Agent mode** (에이전트 모드) | LLM이 다중 턴 대화를 통해 점진적으로 작업을 수행하는 방식. 단일 API 호출과 대비되는 개념 |
| **Vibe coding** (바이브코딩) | 개발자가 자연어로 요구사항만 기술하고 LLM에 코드 생성을 전적으로 위임하는 개발 방식. Karpathy(2025)가 명명 |

## 3. 소프트웨어 공학 용어

| 용어 | 설명 |
|------|------|
| **DDL** (Data Definition Language) | 데이터베이스의 테이블, 컬럼, 제약조건 등을 정의하는 SQL 명령어 (CREATE TABLE 등) |
| **API** (Application Programming Interface) | 소프트웨어 간 통신 규약. 본 논문에서는 REST API 엔드포인트를 의미 |
| **REST** (Representational State Transfer) | 웹 API 설계 방식. HTTP 메서드(GET, POST, PUT, DELETE)로 자원을 조작 |
| **HTTP Status Code** | API 응답 상태를 나타내는 숫자 코드. 예: 201(생성 성공), 409(충돌) |
| **SDLC** (Software Development Life Cycle) | 소프트웨어 개발 생명주기. 요구사항 분석 → 설계 → 구현 → 테스트 → 배포 |
| **CRUD** | Create(생성), Read(읽기), Update(수정), Delete(삭제)의 기본 데이터 조작 |
| **Markdown** (마크다운) | 텍스트 기반 경량 문서 작성 형식. ALS 문서의 기본 포맷 |
| **CLI** (Command Line Interface) | 텍스트 명령어로 프로그램을 조작하는 인터페이스. Claude Code CLI가 실험 도구 |
| **Static analysis** (정적 분석) | 코드를 실행하지 않고 소스 코드를 검토하여 로직 존재 여부를 판정하는 방법 |

## 4. 실험/통계 용어

| 용어 | 설명 |
|------|------|
| **Exploratory study** (탐색적 연구) | 가설의 방향성을 탐색하는 초기 단계 연구. 본 논문은 n=5의 탐색적 연구 |
| **Descriptive statistics** (기술 통계) | 평균, 표준편차 등으로 데이터의 특성을 요약하는 통계 방법 |
| **Standard deviation (SD)** | 표준편차. 데이터가 평균으로부터 얼마나 흩어져 있는지를 나타내는 값 |
| **Outlier** (이상치) | 다른 데이터와 크게 동떨어진 비정상적 값 |
| **Ablation study** (절제 연구) | 시스템의 특정 구성 요소를 제거하여 각 요소의 독립적 기여도를 측정하는 실험 방법 |
| **Cohen's kappa** | 두 평가자 간 일치도를 측정하는 통계 지표. 1에 가까울수록 높은 일치 |
| **Confirmation bias** (확증 편향) | 자신의 기대에 부합하는 결과를 선호하는 인지 편향 |
| **Confounding variable** (혼재 변수) | 독립변수 외에 결과에 영향을 미칠 수 있는 통제되지 않은 변수 |
| **Ceiling effect** (천장 효과) | 측정값이 이미 최대치에 가까워 추가 개선의 여지가 없는 현상 |
| **%p** (퍼센트포인트) | 퍼센트 간 차이를 표현하는 단위. 예: 90%에서 96%로 증가하면 +6%p |

## 5. WMS(창고관리시스템) 도메인 용어

| 용어 | 설명 |
|------|------|
| **WMS** (Warehouse Management System) | 창고 관리 시스템. 입고, 출고, 재고 이동, 재고 실사 등을 관리하는 소프트웨어 |
| **PO** (Purchase Order) | 구매 발주서. 공급자에게 상품을 주문하는 문서 |
| **Inbound** (입고) | 외부에서 창고로 상품이 들어오는 프로세스 |
| **Outbound** (출고) | 창고에서 외부로 상품이 나가는 프로세스 |
| **FIFO** (First In First Out) | 선입선출. 먼저 입고된 상품을 먼저 출고하는 원칙 |
| **FEFO** (First Expired First Out) | 유통기한 임박 선출. 유통기한이 가장 빠른 상품을 먼저 출고하는 원칭. 식품/의약품 등에서 FIFO보다 우선 |
| **Over-receiving** (초과입고) | 발주 수량보다 많은 수량이 입고되는 상황. 본 논문에서는 10% 허용 한도 규칙 적용 |
| **Backorder** (미납 주문) | 재고 부족으로 즉시 출고하지 못하고 대기 중인 주문 |
| **Stock transfer** (재고 이동) | 창고 내 또는 창고 간 상품을 이동하는 작업 |
| **Cycle count** (순환 실사) | 전체 재고 조사 대신 일부를 주기적으로 점검하는 재고 실사 방법 |
| **Safety stock** (안전재고) | 수요 변동이나 공급 지연에 대비하여 유지하는 최소 재고 수준 |
| **HAZMAT** (Hazardous Material) | 위험물. 전용 보관 구역이 필요하며 일반 상품과 혼적 금지 |
| **Shelf life** (유통기한) | 상품의 소비/사용 가능 기간. 잔여율(%)로 입고 허용 여부를 판정 |
| **Audit log** (감사 로그) | 모든 거래/변경 이력을 기록하는 추적 데이터. 삭제 불가 원칙 적용 |

## 6. 실험 그룹 구분

| 그룹 | 제공 문서 | 설명 |
|------|---------|------|
| **Group A** | 요구사항 정의서만 | 바이브코딩 대조군. 설계 문서 없이 LLM에 개발을 위임하는 시나리오 |
| **Group B** | 요구사항 + 자연어 설계문서 | 자연어(산문) 형태의 설계 문서를 추가 제공. DDL, API 명세, 비즈니스 규칙을 포함 |
| **Group C** | 요구사항 + ALS 문서 | Group B와 동일한 설계 정보를 ALS 형식으로 구조화하여 제공 |

### 효과 측정 방식
- **정보량 효과 (B - A)**: 설계 문서 유무의 차이
- **구조화 효과 (C - B)**: 동일 정보를 ALS로 구조화한 효과
