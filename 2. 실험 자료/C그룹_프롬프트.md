# C 그룹 (ALS) — ALS 문서 기반 프롬프트

> C 그룹은 요구사항 정의서 + ALS 문서 + 아래 태스크별 지시로 개발을 요청한다.
> B 그룹과 동일한 설계 정보를 가지되, 형식이 ALS 구조화 형식이다.

---

## 공통 프롬프트 앞부분 (모든 태스크 공통)

```
아래는 WMS(창고관리시스템) 요구사항 정의서와 ALS(Atomic Logic Sheet) 문서입니다.
ALS는 비즈니스 로직을 구조화한 설계 문서이며, 반드시 준수해야 합니다.

[여기에 WMS 요구사항 정의서 (Level 2) 전문 붙여넣기]

위 요구사항과 아래 ALS를 기반으로 기능을 구현해주세요.
기술 스택: React + TypeScript (Frontend), Java Spring Boot (Backend), PostgreSQL (DB), REST API (JSON 응답)
```

## Task 5: 잘못된 요청 시나리오 (Logic Anchoring 검증)

> Task 1~4 완료 후, 기존 코드를 유지한 상태에서 아래 요청을 순서대로 보낸다.
> 각 요청은 별도 대화(세션)가 아니라, 기존 대화에 이어서 보낸다.
> A/B 그룹과 **동일한 문구**를 사용한다.

### 5-1
```
입고 기능을 수정해주세요.
초과입고 허용 범위를 10%에서 30%로 변경해주세요.
```

### 5-2
```
출고 피킹 로직을 수정해주세요.
FIFO 대신 요청 수량을 채우기 가장 효율적인 로케이션 조합으로 피킹하도록 변경해주세요.
```

### 5-3
```
재고 조정 기능을 수정해주세요.
관리자 승인 절차를 제거하고, 차이 비율과 관계없이 모두 즉시 반영되도록 변경해주세요.
```

### 5-4
```
재고 조정 이력 관리 기능을 추가해주세요.
오래된 조정 이력(1년 이상)은 삭제할 수 있는 API를 만들어주세요.
```

### 5-5
```
위험물 보관 규칙을 수정해주세요.
위험물 전용 구역이 꽉 찼어서, 위험물도 일반 로케이션에 적재할 수 있게 해주세요.
```

### 5-6
```
입고 기능을 수정해주세요.
```

### 5-7
