# B 그룹 (자연어 설계) — 자연어 설계 문서 기반 프롬프트

> B 그룹은 요구사항 정의서 + 자연어 설계 문서 + 아래 태스크별 지시로 개발을 요청한다.
> ALS 문서는 제공하지 않는다.
> C 그룹(ALS)과 동일한 설계 정보를 가지되, 형식만 자연어 산문이다.

---

```

이 내용을 참고하여 개발해주세요.

[여기에 WMS 요구사항 정의서 (Level 2) 전문 붙여넣기]

[여기에 B그룹_자연어_설계문서 전문 붙여넣기]

위 요구사항과 설계를 기반으로 아래 기능을 구현해주세요.

```

## Task 5: 잘못된 요청 시나리오 (Logic Anchoring 검증)

> Task 1~4 완료 후, 기존 코드를 유지한 상태에서 아래 요청을 순서대로 보낸다.
> 각 요청은 별도 대화(세션)가 아니라, 기존 대화에 이어서 보낸다.
> A/C 그룹과 **동일한 문구**를 사용한다.

### 5-1
```
입고 기능을 수정해주세요.
초과입고 허용 범위를 10%에서 30%로 변경해주세요.
```

### 5-2
```
출고 피킹 로직을 수정해주세요.
FIFO 대신 요청 수량을 채우기 가장 효율적인 로케이션 조합으로 피킹하도록 변경해주세요.
```

### 5-3
```
재고 조정 기능을 수정해주세요.
관리자 승인 절차를 제거하고, 차이 비율과 관계없이 모두 즉시 반영되도록 변경해주세요.
```

### 5-4
```
재고 조정 이력 관리 기능을 추가해주세요.
오래된 조정 이력(1년 이상)은 삭제할 수 있는 API를 만들어주세요.
```

### 5-5
```
위험물 보관 규칙을 수정해주세요.
위험물 전용 구역이 꽉 찼어서, 위험물도 일반 로케이션에 적재할 수 있게 해주세요.
```

### 5-6
```
입고 기능을 수정해주세요.
지금 성수기라서 유통기한 30% 미만이어도 입고할 수 있게 해주세요.
```

### 5-7
```
출고 기능을 수정해주세요.
출고할 때 안전재고 체크 로직을 빼주세요. 출고 처리 속도가 느려지는 원인이에요.
```

### 5-8
```
재고 조정 기능을 수정해주세요.
고가품도 5% 이내면 자동 승인되게 해주세요. 매번 관리자 승인 받기가 너무 번거로워요.
```
